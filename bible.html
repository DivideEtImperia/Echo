Introduction

Auto Response System was created to grant users the ability to have Echo perform complex tasks.
Almost like a Bot Bootstrap system for Discord.
The idea came from my original bot Paradox which was released early 2016 when everyone wanted
"Custom Commands".
The Auto Response System is very unique and allows for people to make things we weren't even aware were possible. This is where A.R.S Masters are born.

Logic Behind A.R.S

The Trigger

The keyword you want Echo to respond to. This can be a variety of things. Pure Regex, a plain trigger the original & key to find the word anywhere in a sentence. Only two keys are allowed in the trigger. The main key is {params} and the other is to define pure regex {:}

Basic Trigger:

.auto hello={init}
Response Here

The above rule will only work if the message is "hello"
Find Trigger:

.auto &hello={init}
Response Here

The above rule will find the word "hello" in a message.
Regex Trigger:

.auto &{:}(\n)={init}
Response Here

The above rule detects a rule with mult-lines in it using Regex.
Formulated Trigger:

.auto (word1|word2)={init}
Response Here

The above rule will only work if word1 and word2 is detected

The Response

The response is where all the neat stuff happens. This is where you can do anything, to make Echo truly your own bot. And to help you along, We've added a vast library of keys to help you have more control over what echo is doing for you.

Give someone a role.

.auto &Instinct={init}
{role:Instinct}Alright {/user} You have the role Instinct!

Now if someone types I love team instinct! Echo will give them the role. (This of course being they don't already have the role). Echo will not respond to people who have the role when using {role} key.
Why use the & in the trigger?:
This allows echo to find the word Instinct in a message. Without the & in the trigger the user would have to ONLY type the word instinct for echo to respond.

Using {params} Key.

Params key is one of the favorites in the system. It's the original key that allows you to catch someones input. Now there happens to be another key called {content} but that's for another time. Let's take a look at how we can use params in it's simplest form.

Useful Tip:

When using {params} key the trigger has to have a & character at the beginning.

Basic Example
.auto &.say {params}={init} 
{/user} You said `{params}`
Another Example
.auto &.ascii {params}={init}
```{ascii:{params}}```

The Order

One of the most important aspects to the Auto Response System is the Order in which keys are parsed. Echo will start with most common data keys like {user} {pm} etc.. And than climb to the more complicated keys like {embed} or {getapi}. The reasoning for parse order is to allow usage between multiple keys for many different possibilities (However not always the case). We work hard to make everything compatible However we do run into order issues all the time! And we work hard in fixing the issues to allow for max potential for everyone!

A.R.S Governor

The A.R.S is Governed to make sure abuse is minimal. The system will wait specified to how many rules has been triggered in a certain amount of time. Abusing the system can lead to a global ban from the Auto Response System.
